# Work Log

Dated session entries that capture **why** each piece of work was done, **what triggered it**, and **what's left** — so picking the project back up after a week (or longer) is cheap.

- One file per session: `YYYY-MM-DD-<feature>.md`, where `<feature>` is a short kebab-case slug of what the session was about (e.g. `2026-06-28-log-viewer.md`). A `-2`, `-3` suffix is added on an exact name collision.
- Date-first naming means filenames sort chronologically, and the slug makes the listing self-describing at a glance.
- Maintained by the `/worklog` skill: run `/worklog recap` when you sit down, `/worklog wrap` before you stop.

These are committed on purpose — the rationale travels with the code and shows up in `git log`.
